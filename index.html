Executive Summary

Photoshoot Tracker Module — What, Why & Impact

The Photoshoot Tracker is a purpose-built internal portal that digitises and automates the end-to-end lifecycle of own-brand product photography — from stock identification through to shoot completion and automatic bin reversion in AX.
🎯

The Problem

Own-brand product shoots managed entirely manually — scattered stock, no system visibility, and error-prone Excel tracking create delays and coordination failures.

🎯
⚙️

The Solution

A centralised portal that auto-populates eligible products from AX, automates bin-to-bin transfers via the AX API, and provides real-time shoot status to all stakeholders.

⚙️
📈

The Impact

Eliminates 5 manual steps per product, removes Excel dependency, reduces stock displacement risk, and delivers a full audit trail with live visibility across all teams.

The Solution

Auto-Population Logic, Smart Entry & Portal Scope

Auto-Population Criteria

A product appears in the portal automatically — no manual entry required — only when all three conditions are met:

1
AX CatalogThe product exists in the AX ERP system
2
"own_product" Attribute = TRUEAttribute flag set to TRUE in AX
3
Free Stock ReceivedInitial free stock assigned and received in AX
✓  No manual data entry — portal is always in sync with AX
Exception: "Not Required" Override

If a product already has a final approved image, the operator clicks "Not Req." This action:

Flags the item directly in AX
Excludes it from the portal list permanently
Does NOT trigger any bin transfer
📌  Scope Boundary
Stock logistics & status tracking only.Final image upload to the product catalogue remains a manual task — an intentional boundary keeping the module focused and reliable.

Automated Workflow

End-to-End State Machine — Stock Arrival → Shoot Complete

StatusTrigger / User ActionSystem Automation
Awaiting PhotoshootOperator clicks [Assign to Photoshoot]→Instantly creates a new bin-to-bin transfer order in AX. Stock moves from free-stock bin to the photoshoot studio bin automatically.
In ProgressPhysical stock transfer completes in AX→System shifts status to "In Progress". Action button updates to [Shoot Complete]. All stakeholders see live status instantly.
Shoot CompleteOperator sets status to "Shoot Complete"→Initiates a JSON call to the AX API — stock automatically returned from shoot bin to its original bin. [Assign] button disabled permanently.
Not RequiredOperator clicks [Not Req.]→Product flagged in AX and excluded from portal list. No bin change triggered — product already has an approved image.
